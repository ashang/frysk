Frysk

Building Frysk

These notes assume that frysk is being built on a Fedora Core 4 i386 system.

Building on other GNU/Linux systems is certainly possible. People have managed to build frysk on both Red Hat Enterprize Linux 4 and Fedora Raw Hide.

A Frysk RPM for Fedora Core 4 is available that will install a snapshot of Frysk. These are updated a few times a week to reflect new changes. You may want to build frysk from CVS to get the newest version, and this will still require getting all of Frysk's requirements.

Install additional standard RPMS
Install the following RPMs from the standard distribution (note that they have dependencies, and the dependencies should also be installed): 
java-1.4.2-gcj-compat-devel
junit
antlr
transfig
You can do this by entering the command: 
yum install java-1.4.2-gcj-compat-devel junit antlr xfig
Download and install the non-standard RPMS
You can do this by downloading the following yum repository configuration file and copying it into the /etc/yum.repos.d/ directory (you'll need to be root). Then run: 
yum --exclude '*debuginfo' install 'frysk-*'
This will install all the non-standard packages needed by Frysk, but not Frysk itself. In the future you can update the individual packages through yum, or just run yum update 'frysk*'. The quotes are important, otherwise the shell eats away the * and yum gets confused.

Installing the Frysk RPM
If you would like to see it's latest state of Frysk without the hassle of building from source you can install the Frysk RPM. This is not a released version, but shows a recent snapshot of the development code-base. The RPM can be installed with: yum install frysk . All of the requirements should be pulled in by yum automatically if you have not yet installed them.

Command Line Build
Check out frysk:
cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/frysk co frysk
Build the configuration files: 
( cd frysk && ./autogen.sh )
Create a separate build directory, and build in there:
( mkdir build && cd build && ../frysk/configure && make)

Eclipse Build
For eclipse, check out the cvs module frysk-eclipse.

Other Systems

If you are interested in trying frysk on other distributions, or cannot use our prepackaged required dependencies then you will need to find packages for your distribution or compile all the required packages from source. Some of the dependencies for Frysk are in development or newly released (i.e. Java-GNOME packages, GTK+ stack), therefore it may be difficult to find these in pre-packaged form if you are not able to use our supplied packages. Others, such as antlr or JUnit, may be easier to find. You can build these packages from source, and frysk should detect their existance in the standard /usr or non-standard /opt. However, please note that installing the whole stack from source is a formidable task.

For the brave here are the absolutely required packages and versions:

  • antlr 2.7.4
  • JUnit (version: to be determined)
  • GCJ 4.0.0
  • A 1.4.2-compatible Java compiler ( ECJ is strongly encouraged for compatibility reasons)
  • transfig 3.2.4
  • jdom 1.0.0
  • frysk-cdtparser 3.0.0 (this is an extracted binary compiled version of the CDT parser).
  • fontconfig 3.2.3
  • cairo 1.0.0
  • pango 1.10.0
  • GTK+ 2.8.0
  • VTE 0.11.11
  • jg-common 0.1
  • cairo-java 1.0.0
  • libgtk-java 2.8.1
  • libglade-java 2.12.4
  • libvte-java 0.11.11

Please note that some of these packages may have dependencies. Based on our testing on FC4, the above are the bare minimum packages that are needed, however this may differ on your distribution based on what versions are installed.
