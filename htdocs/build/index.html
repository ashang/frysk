Building Frysk

Preparation

  • It is assumed that you're running Fedora Core 4 on an i386 platform.
  • Install the following RPMs from the standard distribution (note that they have dependencies, and the dependencies should also be installed):
  • java-1.4.2-gcj-compat-devel
  • junit
  • antlr
  • Install jdom from the fedora-devel repository. You can do this by entering the command yum --enablerepo=development install jdom. It is highly recommended that you only install the jdom package from development, all dependancies should be installed with the development repository disabled.
  • Download and install the all of the non-standard RPMS. You can do this by downloading the following yum repository configuration file and put it into /etc/yum.repos.d/ (you'll need to be root). Then run yum install frysk-cdtparser frysk-libglade-java. This should install all the needed packages. In the future you can update the individual packages through yum.

Command Line Build

Check out frysk:
cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/frysk co frysk
Build the configuration files:
( cd frysk && ./autogen.sh )
Create a separate build directory, and build in there:
( mkdir build && cd build && ../frysk/configure && make)

Eclipse Build

For eclipse, check out the cvs module frysk-eclipse.

Other Systems

If you are interested in trying frysk on other distributions, or cannot use our prepackaged required dependencies then you will need to find packages for your distribution or compile all the required packages from source. Some of the dependencies for Frysk are in development or newly released (i.e. Java-GNOME packages, GTK+ stack), therefore it may be difficult to find these in pre-packaged form if you are not able to use our supplied packages. Others, such as antlr or JUnit, may be easier to find. You can build these packages from source, and frysk should detect their existance in the standard /usr or non-standard /opt. However, please note that installing the whole stack from source is a formidable task.

For the brave here are the absolutely required packages and versions:

  • antlr 2.7.4
  • JUnit (version: to be determined)
  • GCJ 4.0.0
  • A 1.4.2-compatible Java compiler ( ECJ is strongly encouraged for compatibility reasons)
  • fontconfig 3.2.3
  • cairo 1.0.0
  • pango 1.10.0
  • GTK+ 2.8.0
  • jg-common 0.1
  • cairo-java 0.9.3
  • libgtk-java 2.7.0
  • libglade-java 2.11.4

Please note that some of these packages may have dependencies. Based on our testing on FC4, the above are the bare minimum packages that are needed, however this may differ on your distribution based on what versions are installed.
