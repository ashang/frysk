Building Frysk

  • It is assumed that you're running Fedora Core 4 on an i386 platform.
  • Install the following RPMs from the standard distribution (note that they have dependencies, and the dependencies should also be installed):
    java-1.4.2-gcj-compat-devel
    junit
    antlr
  • Download and install the following non-standard RPMS (they will be installed into /opt):
    frysk-cairo-1.0.0-1.i386.rpm
    frysk-cairo-debuginfo-1.0.0-1.i386.rpm
    frysk-cairo-devel-1.0.0-1.i386.rpm
    frysk-cairo-java-0.9.3-1.i386.rpm
    frysk-cairo-java-0.9.3-2.i386.rpm
    frysk-cairo-java-debuginfo-0.9.3-1.i386.rpm
    frysk-cairo-java-debuginfo-0.9.3-2.i386.rpm
    frysk-fontconfig-2.3.2-1.i386.rpm
    frysk-fontconfig-debuginfo-2.3.2-1.i386.rpm
    frysk-fontconfig-devel-2.3.2-1.i386.rpm
    frysk-glib2-2.8.1-1.i386.rpm
    frysk-glib2-debuginfo-2.8.1-1.i386.rpm
    frysk-glib2-devel-2.8.1-1.i386.rpm
    frysk-gtk2-2.8.3-1.i386.rpm

    Command Line Build

    Check out frysk:
    cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/frysk co frysk
    Build the configuration files:
    ( cd frysk && ./autogen.sh )
    Create a separate build directory, and build in there:
    ( mkdir build && cd build && ../frysk/configure && make)

    Eclipse Build

    For eclipse, check out the cvs module frysk-eclipse.

    Related information

    Some of the dependencies for Frysk are in development or newly released (i.e. Java-GNOME packages, GTK+ stack), therefore it may be difficult to find these in pre-packaged form if you are not able to use our supplied packages. You can build these packages from source, and frysk should detect their existance in the standard /usr or non-standard /opt. However, please note that installing the whole stack from source is a formidable task.
