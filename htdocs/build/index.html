Building frysk From Source

This page provides guidelines for building frysk, checked out from CVS, on a number of systems. In all cases the basic process is:
Build / Install frysk's dependant libraries
You will need to ensure that the following are installed:
junit antlr transfig gcc-java gcc-c++ cvs
See the operating specific sections below for more details
Check out frysk from CVS
cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/src co frysk
Create the configuration files
( cd frysk && ./autogen.sh )
Configure and Build
( mkdir build && cd build && ../frysk/configure && make -j2 )
frysk can then be run from the build tree
./build/frysk-gui/frysk/gui/FryskGui

Red Hat Enterprize Linux 4

frysk requres Java-GNOME bindings, and a number of other libraries, which are not part of the base RHEL 4 distribution. These libraries are are all included in the frysk source RPM.
Build and install the existing RHEL 4 frysk SRC RPM
This RPM provides up-to-date versions of all the libraries that frysk depends on.
See above for normal instructions

Fedora Core 5

All of the RPM's required by frysk are included in Fedora Core 5.

Fedora Core 4

The RPM's that frysk require are not a part of the standard Fedora Core 4 distribution. Additional RPMs need to be downloaded and installed. In addition the gcj compiler has problems.
Install eclipse-ecj
The bundled gcj compiler contains a number of bugs; they can be worked around by using ecj.
Add the frysk yum repository
Copy the file frysk.repo in your /etc/yum.repos.d/ directory (you'll need to be root).
Install the non-standard frysk libraries
Run the command (note the quotes): 
yum --exclude '*debuginfo' install 'frysk-*'
Follow as for a normal build
