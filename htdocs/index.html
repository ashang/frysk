Frysk

Overview

The goal of the frysk project is to create an intelligent, distributed, always-on system monitoring and debugging tool that allows developers and system administrators to monitor running processes and threads (including creation and destruction events), monitor the use of locking primitives, expose deadlocks, gather data and debug any given process by either choosing it from a list or by accepting frysk's offer to open a source code or other window on a process that is in the process of crashing or that has been misbehaving in certain user-definable ways.

frysk is free software and so is generally and freely available as both a research and development platform.

Use Cases

Here are a few examples of how frysk can help:
  • C., a compiler developer, spends a lot of time running the testsuite, constantly monitoring for compiler crashes. For each crash, C. then has to go back and reproduce the failing scenario under a debugger so that they can determine what went wrong. Using frysk, C. can set up:
    • an exit observer that alerts C. that the test compiler is crashing (and gives C. an opportunity to debug it).
    • a trace observer that logs key parts of compiler as it runs
    As soon as the compiler crashes, C. is alerted, and can start analyzing what went wrong.

  • G., a GNU/Linux SysAdmin, is having trouble with the web server. Every so often, but always at 2am, the shopping-cart daemon will repeatedly crash when being started. G., using frysk, can implement a custom observer script that, when a repeated exec/crash is detected:
    • runs a pager to notify G. of the problem
    • snapshot the system
    • saves the trace log
    • disable the shopping-cart
    In addition snapshots and mockups of frysk are available.

    Get the source code
    The frysk source code is made freely available using the standard sourceware-style services.
    Anonymous CVS
    First login to the CVS server as anonymous: 
    cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/frysk login
    {enter "anoncvs" as the password}
    The list of modules can then be listed with. 
    cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/frysk co -c
    More details on how to download and build frysk can be found here.
    Web CVS
    The frysk sources can be browsed using using cvsweb.
    Monitor CVS commits
    CVS commits can be monitored by subscribing to the frysk-cvs mailing list.
