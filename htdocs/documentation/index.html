Recommended Reading

  • Design Patterns: Elements of Reusable Object-Oriented Software
    This book, by the <<gang of four>>, is considered the definitive reference. Some find other more recent equivalent publications easier to digest. The patterns:
    • observer
    • state
    • builder
    are used by key frysk components so understanding them is important.
  • Refactoring: Improving the Design of Existing Code
    Both describes the concept of behavior preserving transformations, and provides a set of refactorings. See also http://refactoring.com/
  • Thinking In Java, 3rd Edition
    An excellent book for learning Java 1.4.2. The 4th edition, which is focused more on Java 1.5 is less useful.
